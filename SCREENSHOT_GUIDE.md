## Capture Checklist

Before each screenshot:
- [ ] Verify correct URL
- [ ] Confirm proper login state
- [ ] Check browser zoom is 100%
- [ ] Clear notifications/toasts
- [ ] Check data is loaded (no spinners)
- [ ] Verify language setting (EN or ES)
- [ ] Remove browser dev tools if open

## Tools & Tips

### Recommended Tools
1. **Greenshot** (Windows) - Free, with annotation
2. **Snagit** (Mac/Windows) - Professional, paid
3. **Browser DevTools** - Built-in screenshot (F12 → Cmd+Shift+P → "Capture full size screenshot")

### Quick Tips
- Use incognito/private window to avoid extensions
- Disable browser extensions that modify pages
- Use same browser for all screenshots (consistency)
- Capture at the same time of day (if timestamps visible)
- Take 2-3 shots of each screen (backup)

## Post-Processing

### Optional Enhancements
1. **Crop to relevant area** (remove unnecessary browser chrome)
2. **Add subtle shadows** (for presentation slides)
3. **Highlight key features** (arrows, circles - use sparingly)
4. **Consistent naming** (e.g., `01_homepage_hero.png`)

## Troubleshooting

### Common Issues

**Issue**: Page shows "No data" or empty states
- **Solution**: Run demo seed functions again, verify data exists

**Issue**: Authentication errors
- **Solution**: Clear cookies, try password reset, check account exists

**Issue**: Pages don't load
- **Solution**: Check browser console for errors, verify routes exist

**Issue**: UI looks broken
- **Solution**: Clear cache, hard refresh (Ctrl+Shift+R)
